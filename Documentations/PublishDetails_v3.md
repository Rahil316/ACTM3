## Full Description

### The actual problem

You already know which design system you want. Material Design 3. Tailwind. Shadcn. You've seen it work. But getting it into Figma — with your colors, your naming, your dark mode, and accessibility that actually holds — is a completely different job.

You're juggling spreadsheets. Checking contrast ratios in a browser tab. Rebuilding the whole thing when a brand color changes. And somewhere along the way, you rename something and a hundred components break.

That is the job Token Wand replaces.

**At a glance:**

- **Start from a real design system** — start from scratch or pick a populer design system; swap in your brand colors and it's fully yours — nothing locked down
- **Accessibility you can't break** — every color in your system is calculated to meet a readability target you set; you will not get a low-contrast warning if you use the plugin right
- **Light, dark, any mode** — add as many color themes as you need; they all generate together in one click
- **Text always reads against the right background** — a button label contrasts against the button, not the page; each color role can have its own context
- **Rename anything, break nothing** — change a color name, restructure your system, reorder everything — Figma variables update in place, no broken components
- **Dev gets the files immediately** — export CSS, SCSS, Tailwind, Swift, Android, React Native, JSON and more directly from the plugin; no spreadsheet, no translation layer
- **Zero lock-in** — everything Token Wand creates are native Figma variables; remove the plugin tomorrow and your file works exactly the same

---

### What it does

Pick your brand colors. Tell the plugin what roles you need — Background, Button, Text, Border, whatever fits your system. Choose your themes. Click Run.

Token Wand writes the entire Figma variable structure directly into your file. Every color, every theme, every variation — named the way you configured, connected correctly, accessible by design. Your developer can export code-ready files the moment you're done.

When a brand color changes: update one hex value, click Run. Everything updates. Nothing breaks.

---

### Accessibility built in, not bolted on

Most tools check accessibility after the fact. Token Wand builds it in from the start.

Every color your system outputs is mathematically calculated to meet the readability level you set — not estimated, not approximately right. If you say "this text needs to be readable on this background," the plugin finds a color that guarantees it, in every theme.

A button label doesn't just need to read against the page — it needs to read against the button. Token Wand understands this. You tell each role what it sits on, and the math accounts for that. Done before a single variable is written.

You will not see a low-contrast accessibility warning. Not because the plugin ignores them — because the colors it produces cannot fail.

---

### Start from what you already know

The built-in Theme Shop has presets for the most common design systems — Material Design 3, Tailwind CSS, Shadcn/Radix, IBM Carbon, Shopify Polaris, Apple HIG, and more. Each one is a complete, ready-to-run setup, not just a color palette.

Load a preset, swap in your colors, run.

The important difference from a community Figma library: this one is yours. Change how many color steps you want. Rename any role. Adjust a readability target. Add a high-contrast mode for enterprise clients. Everything recalculates. No manual checks. No broken references. No locked-down structure you have to work around.

---

### Made to last

Color systems break when things change — a rename, a rebrand, a restructure. Token Wand is built around the assumption that things will change.

Every color and role has a stable internal identity. Rename "Primary" to "Brand/Primary." Reorder your roles. Restructure the whole naming convention. The next time you run, Figma renames the existing variables in place. Every component that referenced them still works.

For teams who want full control: the plugin also generates a structured color range per brand color (light to dark, with steps), so designers and developers share the same foundation. If you prefer token-only output without the full range, that works too — Token Wand can solve each color directly without generating a palette at all.

---

### Dev handoff, done

The plugin exports directly in the formats developers actually use.

CSS custom properties, SCSS, Tailwind config, DTCG JSON, Style Dictionary, Swift, Android XML, React Native, CSV audit sheet — all from within the plugin, no plugins on top of plugins.

What's in your Figma file is what ships in code.
